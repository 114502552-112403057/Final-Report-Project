# 期末分組專案報告：開源專案評估與貢獻實戰

## 專案簡介與目的

本 Repository 存放「文件修正與優化」小組的期末專題報告。

本專案依據課程要求，對知名開源專案 **`yt-dlp`** 進行深入評估，並實際參與其開源協作流程，以實踐性的貢獻為目標，詳細記錄從問題發現到 Pull Request 最終合併（Merge）的完整過程。

## 核心內容

### 1. 專案評估方法：A.C.T.I.V.E 指標

我們採用 **A.C.T.I.V.E** 評估模型，從六個面向（活躍度、溝通程度、透明度、Issue 整潔度、版本穩定性、貢獻易用性）對目標專案進行量化分析：

| 指標代號 | 指標名稱 | 評估內容 |
| :--- | :--- | :--- |
| **A** | Activity（活躍度）| Commit、PR、Issue 更新頻率 |
| **C** | Communication（溝通程度）| 維護者的回覆速度 |
| **T** | Transparency（治理透明度）| 專案規範完整度 (CONTRIBUTING/CONDUCT) |
| **I** | Issue | 標籤分類清晰度與覆蓋率 |
| **V** | Version Stability（版本穩定性）| Release 規律度與穩定性 |
| **E** | Ease of Contribution（貢獻易用性）| 上手難度、文件完整度 |

### 2. 評估結果摘要

* **目標專案：** **[yt-dlp](https://github.com/yt-dlp/yt-dlp)** (影音下載 CLI 工具)
* **結論：** yt-dlp 具備高度的專案健康度與透明度，是**強烈推薦**的開源貢獻目標。
